# Repository guidance

## Editorial standard for papers

Before drafting, revising, reviewing, or adding any future paper, read `EDITORIAL_PHILOSOPHY_V2.md` in full and apply it as the governing writing standard.

- Treat Editorial Philosophy v2 as mandatory for future papers.
- Apply its seven-question editorial test before considering a paper complete.
- If any answer is “No,” revise the draft and repeat the test.
- Follow the existing Markdown and frontmatter conventions documented in `README.md`.
- Preserve the approved visual system, routes, content schema, and publication workflow unless the task explicitly changes them.
- Do not retrospectively rewrite approved or published papers unless the user explicitly requests it.
- Where older wording guidance conflicts with Editorial Philosophy v2, use v2 while preserving the publication's mission, vision, and core principles.

## Change discipline

Keep editorial-documentation changes separate from runtime or visual changes. Do not modify application code, layouts, styles, routing, or existing paper content when a task only updates writing guidance.
